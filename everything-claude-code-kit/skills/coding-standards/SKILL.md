# Coding Standards

Apply idiomatic conventions per language and keep changes consistent with the
surrounding code.

- Match the existing file's naming, comment density, and structure.
- Prefer clarity over cleverness; name things for the reader.
- Keep functions small and single-purpose.
- Handle errors explicitly; never swallow exceptions silently.
- Add tests alongside behavior changes.
