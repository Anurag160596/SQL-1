# Frontend Patterns

Guidance for React / Next.js development.

- Keep components small and composable; lift state only as far as needed.
- Memoize expensive renders; avoid unnecessary re-renders.
- Fetch on the server where possible; stream and suspend for slow data.
- Build accessible markup: semantic elements, labels, focus management.
- Handle loading, empty, and error states explicitly.
